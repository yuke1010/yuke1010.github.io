Survival Analysis and Text-to-SQL Experiments on Telco Customer Churn Data

By yuke1010 | Statistics and Data Science

Background

This project investigates customer churn from two complementary perspectives. First, I treated churn as a time-to-event problem and applied survival analysis methods. Second, I constructed a MySQL database and evaluated whether an LLM could generate correct SQL for analytical questions.

Dataset

The analysis used the IBM Telco Customer Churn dataset. The original dataset contains 7043 records. For the survival analysis section, I followed the official tutorial and focused on customers with month-to-month contracts and active internet service.

Survival Analysis

I implemented Kaplan–Meier estimation, Cox proportional hazards modeling, and several AFT models. The Kaplan–Meier curve showed a clear decline in retention probability as customer tenure increased. The overall median survival time in the filtered subgroup was about 34 months.

The Cox model indicated that variables such as online backup and tech support were associated with lower churn hazard, while paperless billing was associated with slightly higher churn risk. I also compared Weibull, Log-Normal, and Log-Logistic AFT models, and the Log-Normal model achieved the best fit according to AIC.

Text-to-SQL Experiments

I then built a MySQL database and tested LLM-generated SQL queries. The model was able to answer some simple questions correctly, such as grouped averages and grouped counts. However, it also failed on more complicated tasks.

The failure cases revealed several typical problems:

  • misunderstanding the value semantics of categorical columns,
  • generating corrupted or unstable structured output for complex analytical SQL,
  • using an incorrect subgroup baseline in nested aggregation queries.

Reflection

These experiments show that LLMs can be useful assistants for drafting SQL, but they should not be treated as fully reliable analytical tools without human verification. On the other hand, survival analysis provides a principled statistical framework for understanding churn over time and gives more interpretable results than a simple binary view of churn.

Conclusion

Overall, this project combined statistical modeling, database construction, and LLM-based query generation in one workflow. It helped me better understand both the strengths and limitations of modern data analysis pipelines.
